# Lessons Learned

| 상황 | 실수 | 교훈 | 방지 규칙 |
|------|------|------|-----------|
| 균등배분 감지 | `isShortKorean`에 2자 한글 포함 → "중동 사태 대응" 같은 정상 2자 단어를 균등배분으로 오판해서 공백 삭제 | 균등배분은 **1자 개별 배치**만 해당. 2자 단어는 단어 간 공백이지 균등배분 아님 | `detectEvenSpacedItems`의 isShortKorean은 `/^[가-힣]{1}$/`만 |
| 한글 갭 기준 | 0.35→0.5로 올렸다가 단어 간 공백이 사라짐 (공백누락 32→51 악화) | 갭 기준 변경은 균등배분과 단어 공백의 **트레이드오프** — 균등배분은 좌표 기반으로 별도 처리하고 갭 기준은 보수적으로 유지 | 갭 기준 변경 전 반드시 공백누락 카운트 확인 |
| pdfplumber 과대평가 | pdfplumber로 교체하면 다 해결될 거라고 잘못 판단 | pdfplumber도 pdfminer.six 기반으로 **동일한 균등배분/CJK 공백 문제** 있음. 한국 공문서 특화 기능은 kordoc이 유일 | 외부 라이브러리 교체 권고 전 해당 라이브러리의 CJK/한국어 이슈 실제 조사 필수 |
| 정규식 후처리 vs 좌표 기반 | 정규식으로 균등배분을 처리하려다 모든 경우의 수를 커버 못 함 | **근본 해결은 좌표 기반** — `normalizeItems`에서 TextItem 분해 후 `detectEvenSpacedItems`로 처리. 정규식은 안전망으로만 | 텍스트 후처리보다 아이템 좌표 단계에서 처리 우선 |
